TP Prise en main de NetBeans

Un environnement de développement

Dans cette séance de TP seront introduites des notions de Java et l'utilisation de NetBeans. Java est le langage de programmation sur lequel est basé le langage Processing, que vous avez utilisé jusqu'alors, uniquement dans l'environnement Processing. Alors que Processing est à la fois un langage ET un environnement de développement (éditeur graphique avec un bouton de lancement, un mode debug, etc),

  • Java est uniquement un langage de programmation et
  • NetBeans uniquement un environnement de développement intégré (IDE en anglais).
En général, un IDE est adapté à quelques langages seulement. Pour Java, il existe aussi Eclipse ou IntelliJ IDEA (entre autres).

Un IDE est un outil de développement qui simplifie et accélère l'édition du code, et intègre d'autres outils utiles au développement comme la recherche de bug, les tests automatiques, la gestion de dépendances ou encore le versionnement de source. C'est ce que vous allez découvrir avec NetBeans.

Pour vos premiers pas en Java pur, vous allez de nouveau travailler avec des tableaux.

De Processing à Java

Une grande partie de ce que nous avons utilisé avec Processing est utilisable à l'identique en Java pur :

  • Les types de base (
    int
    , 
    float
    , 
    char
    , 
    boolean
    ), leurs opérateurs, les déclarations de variables et l'affectation
  • Les structures conditionnelles (
    for
    , 
    while
    , 
    if/else
    , 
    switch/case/break
    )
  • Les déclarations de fonctions, le type 
    void
    et le mot-clé 
    return
  • Les tableaux, déclarés avec 
    type[]
    , initialisés avec 
    new
    et accédés avec 
    []
    int[] nbJours = new int[12]; // Déclaration et initialisation d'un tableau de 12 entiers
nbJours[1] = 28; // Changement de la valeur de la deuxième case
for (int i = 0; i < 12; i++) { faireQuelqueChose(nbJours[i]); }
  • Les classes avec leurs attributs et constructeurs, ainsi que le passage de référence : 
    class Salle {
  int capacite;
  boolean salleMachine;
  boolean libreService;
  boolean occupee;
  int numero;
} // Définition
...
static void reserverSalle(Salle s) { s.occupee = true; } 
...
static void maFonction() {
  Salle salle1; // Déclaration d'une variable
  salle1 = new Salle() // Initialisation d'une instance
                       // avec le constructeur par défaut      
  reserverSalle(salle1); // Changement de la valeur d'un attribut
                         // par passage de référence
}
  • Dans ce TP nous utiliserons le mot clé 
    static
    devant toutes les fonctions, comme ci-dessus. Nous verrons pourquoi dès la semaine prochaine.
  • Le mot clé 
    final
    , utilisé dans la déclaration d'une variable pour empêcher son changement de valeur. 
    final int NB_SALLES = 42; // En général on écrit ces variables en majuscules.

Certaines choses changent :

  • En Java, n'importe quel code se trouve forcément dans une classe. Alors qu'en Processing on pouvait mettre du code en dehors d'une classe.
  • En Java il n'y a pas de classe "principale". Par contre certaines classes contiennent une fonction 
    main()
    qui permet de lancer cette fonction comme un programme.
  • Pour les affichages, 
    println()
    est remplacée par 
    System.out.println()
    , tout comme 
    print()
    .
  • Les fonctions de dessin (
    fill()
    , 
    rect()
    , 
    ellipse()
    , etc). qui font partie de la bibliothèque Processing ne sont plus disponibles.
  • Les fonctions 
    setup()
    , 
    draw()
    et 
    mouseXXX()
    n'existent pas non plus.

Compilation et exécution

Dans la barre principale, au dessus de l'onglet d'édition, l'icône marteau ⚒ vous permet de compiler votre projet (aussi F11), l'icône lecture ‣ permet de lancer l'exécution (aussi F6).

La compilation permet de vérifier la validité du code écrit, et de générer un programme exécutable. Un message d'erreur sera produit si la syntaxe n'est pas bonne. Par défaut, l'IDE NetBeans compile automatiquement le projet à chaque enregistrement de modification (Ctrl+S). Enregistrer et recompiler régulièrement permet de sauvegarder son travail, mais aussi de corriger pas à pas les potentielles erreurs de frappe.

L'exécution d'un programme Java débute toujours par l'appel d'une fonction spéciale dont le prototype/signature est : 

public static void main(String[] args)
Dans notre squelette de programme, cette fonction est déjà déclarée/définie. Le paramètre de cette fonction est un tableau de chaînes de caractères (type 
String
). Chacune de ces chaînes de caractères est un argument en ligne de commande. Ces arguments permettent aux utilisateurs d'un programme de changer son comportement sans avoir à le recompiler. Par exemple, le programme 
head
prend en argument un fichier mais permet aussi à ses utilisateurs de spécifier le nombre de lignes à afficher (avec l'argument 
-n 10
par exemple).

  • Ajouter dans le corps de la fonction 
    main
    un affichage : 
    public static void main(String[] args) {
        System.out.println("Hello World");
}
    Le raccourci Ctrl+Espace vous propose automatiquement une/des complétion(s).
  • Compiler puis exécuter le programme.
  • Repérer le panneau d'affichage de la sortie des programmes (où s'affiche votre message).
  • Introduisez une ou deux erreurs dans le code pour observer les messages d'erreurs après compilation.

Tout en réalisant ce programme, nous allons utiliser différentes fonctionnalités de manipulation de code et de debug de NetBeans.

Remarque sur les tableaux : dans ce TP nous utilisons la classe TableauEntiers pour stocker les entiers. Celle-ci contient un tableau de taille fixe, ainsi que l'attribut taille pour indiquer combien d'éléments sont présents dans le tableau (dans ses premières cases). En Java, on peut connaître la taille d'un tableau via son attribut length : 

int[] monTableau = new int[10];
System.out.println("longueur = " + monTableau.length); // affiche : longueur = 10
Dans ce TP, nous pourrons par exemple l'utiliser pour éviter un dépassement de capacité.
